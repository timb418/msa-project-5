## Контекст задачи

Онлайн-магазин каждое утро в 6:00 должен генерировать кастомные CSV прайс-листы для B2B-клиентов на основе текущих данных в PostgreSQL.

**Данные в PostgreSQL:**

| Таблица          | Объём строк     |
|------------------|-----------------|
| `products`       | 5 000–10 000    |
| `categories`     | 50–200          |
| `clients`        | 100–500         |
| `client_prices`  | 10 000–20 000   |

Требуется связать данные из таблиц и выгрузить в CSV-файл. Дополнительная логика обработки данных не нужна. Инфраструктура — микросервисы в облаке на базе Kubernetes.

## Принятое решение

По результатам сравнительного анализа выбран **Kubernetes CronJob** как наиболее подходящее решение для данного кейса.

Подробное обоснование — в [`justification.md`](justification.md).

Верхнеуровневый план реализации — в [`implementation-plan.md`](implementation-plan.md).
